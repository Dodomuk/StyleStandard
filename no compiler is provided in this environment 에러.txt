[SPRING 빌드 에러] 
오늘 VisualStudioCode로 스프링 공부를 하던 중 다른 개발자의 깃허브에서 끌어온 파일을 실행하는 과정에서 에러가 나 몇시간을 애먹게되었다.

해당 에러 문구: 
No compiler is provided in this environment. Perhaps you are running on a JRE rather than a JDK?

찾아보니 메이븐을 통해 배포 받을때 같은 에러가 난다면 자바 실행 도구에 문제가 생긴 것이라고 한다. 이클립스에 연결된 자바실행 도구가 JRE로 되어 있어 JDK로 변경해야 한다고 한다. 근데 막상 들어보니 JRE와 JDK가 정확히 무슨 차이인지 잘 모르겠다. 이 둘은 뭐가 다를까?

JRE란?
먼저 JRE는 Java Runtime Environment의 약자로 자바 프로그램을 실행시켜주는 환경을 구성해주는 도구다. 즉 JAVA를 개발할 필요는 없는데, 실행은 시켜줘야 하는 경우에는 꼭 JRE가 있어야 한다. 반면 JAVA 개발시 꼭 필요한 것이 있는데 그것이 바로 JDK다.

즉 나는 지금 자바 프로그램을 실행하는 것 뿐만 아닌 개발을 할 경우까지 염두하고 있기 때문에 JDK로 설정을 해줘야 한다는 말이다. 또한 Maven 배포를 위해서는 JDK에 들어있는 컴파일러 등을 사용해야하기 때문에 JDK가 필요하다.

이클립스의 경우 window -> preferences에 현재 자바 JRE 혹은 JDK를 확인 할 수 있다. 그러나 Visual Studio Code의 경우 다른 방법으로 JDK 를 설정해야 한다.


-------------

1. Visual Studio Code에서 File에 들어가면 Preferences 를 찾을 수 있다.
Preferences -> settings 
settings에서 검색창에 Java_home을 입력하면 settings.json으로 들어갈 수 있는데 
이곳이 바로 우리가 java의 JDK 주소를 설정할 곳이다.



*개인마다 다를수 있다

//settings.json
{
    "explorer.confirmDelete": false,
    "explorer.confirmDragAndDrop": false,
    "window.zoomLevel": -1,
    "editor.fontSize": 16,
    "editor.suggestSelection": "first",
    "vsintellicode.modify.editor.suggestSelection": "automaticallyOverrodeDefaultValue",
    "redhat.telemetry.enabled": true,
    "files.exclude": {
        "**/.classpath": true,
        "**/.project": true,
        "**/.settings": true,
        "**/.factorypath": true
    },
    "workbench.colorTheme": "Solarized Dark"
}

현재 이 코드창에 
workbench.colorTheme쪽에 
"java.home": "C:\\Program Files\\Java\\jdk-11.0.9\\bin"(나의 JDK 위치)
를 넣어주면된다.
※bin까지 넣은 이유는 JDK의 경우 실행 파일(JRE)가 JDK 안에 존재하기 때문이다. JRE를 넣을 경우 bin으로 들어갈 필요 없다.
※visual studio code의 경우 파일 경로를 \\ 두번 넣어야 한다. 


-------------

2.혹시 모르니 환경 변수도 체크해야 한다.

시스템 속성에 들어가 환경 변수로 들어가자.
아랫쪽 시스템 변수(S)라고 써져있는 곳에  있는 변수 JAVA_HOME / JDK_HOME의 값이 올바른 JDK 파일의 주소인지 확인해야 한다.
